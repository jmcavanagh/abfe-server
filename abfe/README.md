# ABFE job bundle (GID4 / 9QDZ)

End-to-end **absolute binding free energy (ABFE)** workflow for a single ligand
pose against GID4 (PDB **9QDZ**). `run.sh` parameterizes the ligand, sets up the
three FEP legs with EasyBFE, runs Amber/pmemd MD on GPU, and analyzes the
results when all legs finish.

This directory ships a ready-made example pose (`v2_1b_1place.sdf`) against the
prepared receptor (`9qdz_fixed_dry.pdb`).

## What ABFE computes

ABFE estimates the binding free energy of one ligand to a protein via a
thermodynamic cycle with three legs:

| leg | system |
|-----|--------|
| **complex** | ligand decoupled in the protein binding site |
| **solvent** | ligand decoupled in explicit water |
| **restraint** | Boresch orientational restraints turned on/off in the complex |

When all three legs complete, EasyBFE combines them into ΔG_bind (reported in
`abfe/result.json`).

## Directory contents

| file | role |
|------|------|
| `run.sh` | Job driver: pargen → setup → MD → analyze |
| `config_abfe.yml` | AmberAbfeConfig (workflow, lambdas, force fields) |
| `9qdz_fixed_dry.pdb` | Prepared GID4 receptor |
| `v2_1b_1place.sdf` | Example ligand pose (3D SDF) |

`ligand/` and `abfe/` are **created at runtime** by `run.sh`. Do not check them
in or upload them as job inputs.

## Requirements

Run on a machine with:

- **EasyBFE** CLI (`easybfe`)
- **Amber 26** `pmemd.cuda` and `pmemd.cuda.MPI`
- **NVIDIA GPU(s)** with a working CUDA driver
- **`nvidia-cuda-mps-control`** — `run.sh` starts CUDA MPS so many λ-window
  replicas can share the available GPUs

## Usage

Run from this directory with defaults:

```bash
bash run.sh
```

Override inputs with flags (paths relative to this directory):

```bash
bash run.sh \
  --ligand ./my_pose.sdf \
  --protein ../data/9qdz.pdb \
  --config ./config_abfe.yml
```

Short flags: `-l`, `-p`, `-c`. Run `bash run.sh --help` for the full option list.

| option | default |
|--------|---------|
| `--ligand` / `-l` | `./v2_1b_1place.sdf` |
| `--protein` / `-p` | `./9qdz_fixed_dry.pdb` |
| `--config` / `-c` | `./config_abfe.yml` |

## What `run.sh` does

1. **`easybfe ligand pargen`** — GAFF2 parameters and AM1-BCC charges → `ligand/`
2. **`easybfe abfe setup`** — three leg trees + Boresch restraints → `abfe/`
3. **CUDA MPS** — started before MD, stopped on exit
4. **Run each leg** — `abfe/complex`, `abfe/solvent`, `abfe/restraint` (each leg
   has its own `run.sh` generated by setup). Legs are skipped if they already
   have `done.tag`, `error.tag`, or `running.tag`.
5. **`easybfe abfe analyze`** — runs when all three legs have `done.tag`

Each leg runs EM → heat → pressure equilibration → pre-production → production
(H-REMD across λ windows), as defined in the YAML config.

## Outputs

After a successful run:

```text
ligand/                          # parameterized ligand (topology, coords)
abfe/
  boresch.dat
  complex/   solvent/   restraint/   # per-leg MD outputs, run.log, done.tag
  analyze.log
  result.json                      # binding free energy and leg breakdown
  convergence.csv
  convergence.png
```

Monitor progress:

```bash
ls abfe/*/done.tag
tail -f abfe/complex/run.log
```

## Config file

**`config_abfe.yml`** documents AmberAbfeConfig fields inline (box, ions,
workflow stages, λ schedule, force fields). Adjust `workflow`, `num_lambdas`,
and stage lengths there; do not edit generated files under `abfe/` by hand
unless you know what you are changing.
